Gradient Boosting for classification.¶

https://scikit-learn.org/stable/modules/ensemble.html#gradient-boosting

O Gradient Boosting ou Gradient Boosted Regression Trees (GBRT) é uma generalização de estímulo para funções de perda arbitrariamente diferenciáveis.

O GBRT é um procedimento pronto e efetivo que pode ser usado tanto para problemas de regressão quanto de classificação.

Modelos de Gradient Tree Boosting são usados em uma variedade de áreas, incluindo ranking de busca na Web e ecologia.

As vantagens do GBRT são:

  • Manipulação natural de dados do tipo misto (= características heterogêneas)
  • Potência preditiva
  • Robustez para outliers no espaço de saída (através de funções robustas de perda)

As desvantagens do GBRT são:

  • Escalabilidade, devido à natureza sequencial do aumento, dificilmente pode ser paralelizado.

O módulo sklearn.ensemble fornece métodos para classificação e regressão através de árvores de regressão com aumento de gradiente.

Para a criação de nossa "Baseline", utilizamos a parametrização inicial do sklearn, levemente modificada, e faremos o tuning gradativo de algumas parametrizações, visando o ponto de vista didático.

  • n_estimators=100,

  • tol=0.0001

Parametrização para nossa baseline:¶

  • n_estimators=100
  • max_depth=3
  • min_samples_split=2
  • min_samples_leaf=1
  • subsample=1.0
  • learning_rate=0.1
  • subsample: 1

O que faremos?¶

  • criaremos nosso classificador: "baseline"
  • tuning de alguns parâmetros (efeito didático) e criaremos nosso 2o classificador: "modelo_1"
  • faremos novo tuning de mais alguns parâmetros (efeito didático) e criaremos nosso classificador final: "modelo_final"
  • visando comparação, utilizaremos também o "classificador default do sklearn" e o "classificador utilizado na etapa anterior de validação dos modelos" #### Obs: visando performance dos experimentos optamos por reduzir a quantidade de parametros testados, ou seja, não foram testadas todas as combinações de parâmetros possíveis. Como se trata de efeito didático, entendemos que objetivo (demonstrar tuning/calibração de modelos) foi alcançado.

#função para visualização de Feature Importances
def plot_feature_importances(clf, clf_name = "", figsize=(8,8), print_table=False, title="Feature Importances"):
    est_name = clf.__class__.__name__ + ": " + clf_name    
    # get importance
    if est_name in ['ub','sbag']:
        imp = np.mean([
                e.feature_importances_ for e in clf.estimators_
