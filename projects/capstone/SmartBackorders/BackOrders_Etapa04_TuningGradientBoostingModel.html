Gradient Boosting for classification.¶

https://scikit-learn.org/stable/modules/ensemble.html#gradient-boosting

O Gradient Boosting ou Gradient Boosted Regression Trees (GBRT) uma generalizao de estmulo para funes de perda arbitrariamente diferenciveis.

O GBRT um procedimento pronto e efetivo que pode ser usado tanto para problemas de regresso quanto de classificao.

Modelos de Gradient Tree Boosting so usados em uma variedade de reas, incluindo ranking de busca na Web e ecologia.

As vantagens do GBRT so:

  • Manipulao natural de dados do tipo misto (= caractersticas heterogneas)
  • Potncia preditiva
  • Robustez para outliers no espao de sada (atravs de funes robustas de perda)

As desvantagens do GBRT so:

  • Escalabilidade, devido natureza sequencial do aumento, dificilmente pode ser paralelizado.

O mdulo sklearn.ensemble fornece mtodos para classificao e regresso atravs de rvores de regresso com aumento de gradiente.

Para a criao de nossa "Baseline", utilizamos a parametrizao inicial do sklearn, levemente modificada, e faremos o tuning gradativo de algumas parametrizaes, visando o ponto de vista didtico.

  • n_estimators=100,
  • subsample=1.0,

  • random_state=None,
  • max_features=None,
  • verbose=0,
  • max_leaf_nodes=None,
  • warm_start=False,

  • validation_fraction=0.1,
  • n_iter_no_change=None,
  • tol=0.0001

Parametrização para nossa baseline:¶

  • n_estimators=100
  • max_depth=3
  • min_samples_split=2
  • min_samples_leaf=1
  • subsample=1.0
  • learning_rate=0.1
  • subsample: 1

O que faremos?¶

  • criaremos nosso classificador: "baseline"
  • tuning de alguns parmetros (efeito didtico) e criaremos nosso 2o classificador: "modelo_1"
  • faremos novo tuning de mais alguns parmetros (efeito didtico) e criaremos nosso classificador final: "modelo_final"
  • visando comparao, utilizaremos tambm o "classificador default do sklearn" e o "classificador utilizado na etapa anterior de validao dos modelos" #### Obs: visando performance dos experimentos optamos por reduzir a quantidade de parametros testados, ou seja, no foram testadas todas as combinaes de parmetros possveis. Como se trata de efeito didtico, entendemos que objetivo (demonstrar tuning/calibrao de modelos) foi alcanado.

#funo para visualizao de Feature Importances
def plot_feature_importances(clf, clf_name = "", figsize=(8,8), print_table=False, title="Feature Importances"):
    est_name = clf.__class__.__name__ + ": " + clf_name    
    # get importance
    if est_name in ['ub','sbag']:
        imp = np.mean([
                e.feature_importances_ for e in clf.estimators_
                ], axis=0)
    elif est_name in ['rus','smt']:
